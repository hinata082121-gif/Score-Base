# Score Base Game Entry Flow

## Overview

Score Baseの試合入力は、全モードで最初に「試合情報登録」を行い、その後に必要な入力へ進みます。試合情報だけ保存した記録は `/games` に表示され、入力状況に応じた「続きから入力」導線から再開できます。

## Mode Flows

- 観戦記録のみ: 試合情報登録 -> 試合結果入力 -> 完了
- 簡易記録: 試合情報登録 -> 試合結果入力 -> 完了
- 詳細スコアブック: 試合情報登録 -> 情報確認 -> ホームスタメン -> ビジタースタメン -> スタメン最終確認 -> 試合開始 -> スコアブック入力 -> 試合結果入力 -> 完了

## Game Info Step

試合情報登録では、試合日、球場、大会・リーグ、天気、ホームチーム、ビジターチーム、応援チーム、公開フラグを入力します。チーム、球場、大会、天気は候補選択と新規入力の両方を維持します。

## Resume Status

`/games` では既存データから入力状況を推定して表示します。

- 試合情報のみ
- スタメン入力待ち
- スタメン確認待ち
- 試合進行中
- 試合結果待ち
- 完了

## Lineup UI

詳細スコアブックのスタメン入力はホームとビジターを別ステップに分けます。各チームの画面は、守備配置グラウンド、ラインアップ表、控え選手一覧で構成します。iPhone SE相当でもページ全体の横スクロールを避けるため、入力対象チームだけを表示し、ラインアップと控え一覧はスクロール領域に収めます。

## Result Step

試合結果入力は全モード共通です。最終スコア、勝敗、試合状態、理由メモ、終了時刻、印象に残った選手、MVP、任意メモを記録します。簡易記録と詳細スコアブックでは、イニング別スコアと投手メモもこの画面で入力します。

## Known Constraints

- RunnerEvent専用の入力・保存UIは未実装です。走者状態は引き続き `PlateAppearance.baseStateBefore` / `PlateAppearance.baseStateAfter` で保持します。
- 試合開始後のスタメン破壊的編集は避けます。代打、守備交代、投手交代は次フェーズです。
- DB-backed保存とlocalStorageゲスト保存の両経路を維持します。

## v0.7.13 Input Rules

- 試合情報の必須項目は、試合日、ホームチーム、ビジターチームです。同一チームはエラーになります。
- 球場は保存済みGameの球場とTeam本拠地を候補化し、新規入力も許可します。
- 大会・リーグは保存済みGameのcompetitionを候補化し、新規入力も許可します。
- 天気は「晴れ、曇り、雨、小雨、雪、屋内、その他」固定で、新規入力は許可しません。
- 最終スコアは入力中は文字列で保持し、空欄にできます。blur時に空欄なら0へ戻し、保存時は空欄を0として扱います。
- 簡易記録は「スコアのみ」と「スコアボード」を切り替えできます。
- 選手記録は投手と本塁打者を登録選手候補から選べます。未登録選手の自由入力fallbackも残します。
- 詳細記録のスタメン候補は、ホームならhomeTeamId、ビジターならawayTeamIdに一致するPlayerだけを表示します。teamId未設定時は登録選手候補を出しません。
